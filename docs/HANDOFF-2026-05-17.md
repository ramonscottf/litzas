## Photo phase next

The public menu only shows approved pizza photos. Pizzas without approved photos render as text-first cards with a quiet "Photo in approval" badge.

The current SpotOn zip is a working handoff artifact, not the final pack to send
to SpotOn. Treat it as final only after every manifest entry has an approved
existing or generated image.

Use `data/menu-photo-manifest.json` as the source of truth. For each pizza:

1. Place the approved image at `assets/images/menu/pizzas/<slug>.jpg`.
2. Update `approvalStatus` to `approved-existing` or `approved-generated`.
3. Keep `spotonExportName` aligned for the SpotOn photo pack.
4. Re-render and rebuild the archive:

```bash
node scripts/render-site.mjs
node scripts/package-spoton-photos.mjs
npm test
npm run check
```

AI generation is ready in `scripts/generate-menu-photos.mjs`, but the first generation attempt hit `billing_hard_limit_reached` on the connected OpenAI account before any pilot image was returned.

Pilot command after billing is cleared:

```bash
node scripts/generate-menu-photos.mjs --all --only=pepperoni,western-bbq,vegetarian
```

Full missing-photo command:

